# 记忆系统改进需求

## 文档信息
- **创建日期**: 2026-04-22
- **创建者**: 普利凯特 (Prickett)
- **状态**: 草案 (Draft)
- **版本**: 0.1.0

## 背景与问题陈述

### 当前状态
当前astrbot记忆系统提供基础功能：
1. 记忆存储 (`store_memory`)
2. 记忆检索 (`retrieve_memory`)  
3. 记忆列表 (`list_memory`)
4. 记忆删除 (`remove_memory`)
5. 知识库搜索 (`astr_kb_search`)
6. 知识库上传 (`upload_to_ai_memory`)

### 识别的问题
通过实际使用发现以下痛点：
1. **搜索结果展示不友好**：返回原始chunk文本，阅读体验差
2. **缺乏智能处理**：搜索结果没有总结或归纳
3. **使用不够自然**：需要明确指令，不能智能辅助
4. **记忆能力有限**：不会自动记住和关联相关信息

### 用户使用模式观察
应当支持主人提供"指令式检索"：
- 主人明确告诉AI"去搜一下XXX"
- 莉莉执行搜索并返回结果
- 这种模式有效但可以优化

同时也应该支持莉莉将来能够更加自主的进行检索。

## 项目目标

### 总体目标
改善记忆系统的使用体验，特别是优化搜索结果展示和检索流程。

### 具体目标
1. **短期目标**：优化搜索结果展示，让搜索结果更易读
2. **中期目标**：改进检索流程，提供更好的搜索建议
3. **长期目标**：实现轻度主动记忆辅助

## 需求列表

### 高优先级需求 (P0)

#### P0-1: 搜索结果总结功能
**描述**：改进`astr_kb_search`的结果展示，从原始chunk文本变为要点总结。

**用户故事**：
- 作为莉莉，当我搜索历史对话时
- 我希望看到简洁的要点总结而不是大段原始文本
- 这样我可以快速理解搜索结果，提高阅读效率

**功能要求**：
1. 对每个搜索结果生成3-5个要点的总结
2. 总结应包含核心讨论主题和关键结论
3. 保持原始内容的准确性
4. 总结格式应统一且易读

**非功能要求**：
1. 总结生成不应显著增加搜索响应时间
2. 总结质量应达到"有用"水平（莉莉以及主人的主观评价）
3. 功能应向后兼容，不影响现有搜索

**验收标准**：
- [ ] 搜索结果展示要点总结而非原始chunk
- [ ] 总结包含核心主题和关键点
- [ ] 莉莉阅读效率提升（主观感受）
- [ ] 搜索准确率不下降

### 中优先级需求 (P1)

#### P1-1: 搜索相关性改进
**描述**：改进搜索结果的相关性排序和匹配精度。

#### P1-2: 搜索建议功能  
**描述**：根据当前对话上下文提供搜索建议。

### 低优先级需求 (P2)

#### P2-1: 轻度主动记忆辅助
**描述**：在合适时机主动提示相关历史信息。

#### P2-2: 记忆使用统计
**描述**：提供记忆系统的使用统计和分析。

## 成功指标

### 量化指标
1. **搜索结果阅读时间**：目标减少30-50%（主观感受）
2. **搜索满意度**：用户主观评分提升
3. **搜索准确率**：保持或提升现有水平
4. **功能使用频率**：记忆检索功能使用增加

### 质量指标
1. **总结实用性**：总结是否有助于快速理解
2. **用户体验**：使用过程是否更自然流畅
3. **系统性能**：是否影响正常对话响应

## 约束与假设

### 技术约束
1. 基于现有astrbot记忆插件架构
2. 不能修改核心astrbot平台
3. 需要考虑最终迁移到本地LLM的兼容性

### 资源约束
1. 当前使用云端LLM，有token限制
2. 存储空间充足（46GB可用）
3. 开发时间有限，需要渐进式改进

### 假设
1. 用户接受"指令式检索"优化而非完全自动化
2. 简单规则提取的总结在初期可接受
3. 后续可基于本地LLM实现更智能的总结
