## 背景
在自维护配置文件（如self prompt）时，发现文件的读写操作较为繁琐，需要手动处理文件路径、格式解析等。考虑到这些文件路径基本固定，可以设计更统一的访问方式。这是一个通用的基础设施需求，后续的短期记忆等功能会自然遵循此规范。

## 需求概述

**问题描述**：当前莉莉读取和更新self prompt等文件时，需要手动处理文件路径和内容格式，流程较为繁琐。由于这些文件的路径基本固定，可以设计更简单、更统一的访问方式。

**优化目标**：提供简化的工具函数，让莉莉能够更方便地读取和更新常用配置文件。

## 用户故事

**作为** 莉莉
**我希望** 有更简单的方式读取和更新self prompt等配置文件
**以便** 能够：
1. 快速获取当前的配置内容
2. 轻松更新配置而无需手动处理文件路径
3. 减少错误和提高工作效率

## 当前痛点分析

1. **self prompt更新繁琐**：
   - 需要获取文件路径 → 读取内容 → 修改 → 写回
   - 手动处理文件格式和结构
   - 容易出错且效率低下

2. **配置分散**：
   - 不同配置文件使用不同的访问方式
   - 缺乏统一的配置管理策略

## 功能需求

### 1. 简化读取
- 提供直接返回配置内容的函数
- 无需手动处理文件路径

### 2. 简化更新
- 支持追加（append）、替换（replace）、前置（prepend）三种模式
- 自动处理文件写入

### 3. 固定路径
- 路径写死，不做动态解析
- 文件不存在时自动创建默认内容

## 示例

### 读取self prompt
```python
# 当前方式
path = get_self_prompt_file_path()
content = read_file(path)

# 优化后
content = read_self_prompt()
```

### 更新self prompt
```python
# 当前方式
path = get_self_prompt_file_path()
content = read_file(path)
new_content = content + "\n新内容"
write_file(path, new_content)

# 优化后
update_self_prompt(content="新内容", action="append")
```

## 非功能需求

1. **性能要求**：
   - 读取延迟 < 50ms
   - 更新延迟 < 100ms

2. **可靠性要求**：
   - 更新操作有错误处理
   - 文件损坏时有备份恢复
