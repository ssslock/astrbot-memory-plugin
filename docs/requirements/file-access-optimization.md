# 文件访问优化需求

## 文档信息
- **创建日期**: 2026-04-24
- **创建者**: 普利凯特 (Prickett)
- **状态**: 草案 (Draft)
- **版本**: 0.1.0

## 背景
在自维护配置文件（如self prompt）时，发现文件的读写操作较为繁琐，需要手动处理文件路径、格式解析等。考虑到这些文件路径基本固定，可以设计更统一的访问方式。这是一个通用的基础设施需求，后续的短期记忆等功能会自然遵循此规范。

## 需求概述

**问题描述**：当前莉莉读取和更新self prompt等文件时，需要手动处理文件路径和内容格式，流程较为繁琐。由于这些文件的路径基本固定，可以设计更简单、更统一的访问方式。

**优化目标**：提供简化的工具函数，让莉莉能够更方便地读取和更新常用配置文件。

## 用户故事

**作为** 莉莉
**我希望** 有更简单的方式读取和更新self prompt等配置文件
**以便** 能够：
1. 快速获取当前的配置内容
2. 轻松更新配置而无需手动处理文件路径
3. 减少错误和提高工作效率

## 当前痛点分析

1. **self prompt更新繁琐**：
   - 需要获取文件路径 → 读取内容 → 修改 → 写回
   - 手动处理文件格式和结构
   - 容易出错且效率低下

2. **路径与对话ID绑定而不是与人格绑定**：
   - 需要使用人格名称而不是当前社交软件的账号来决定路径
   - 需要支持多个不同的配置文件以便将来扩展

## 功能需求

### 1. 简化读取
- 提供直接返回配置内容的函数
- 无需手动处理文件路径

### 2. 简化更新
- 支持追加（append）、替换（replace）、前置（prepend）三种模式
- 自动处理文件写入

### 3. 固定路径
- 路径基于插件存储路径
- 文件不存在时输出warning日志

## 示例

### 流程改进对比

**之前**：莉莉需要先调用 `get_self_prompt_file_path` 获取路径字符串，再用该路径去调用另一个tool（如 `retrieve_memory`）来读取/写入文件。两步tool call才能完成一次操作，且涉及手动处理路径。

**之后**：莉莉直接调用 `read_self_prompt` 或 `update_self_prompt`，一步tool call完成操作，无需关心路径。

## 非功能需求

1. **性能要求**：
   - 读取延迟 < 50ms
   - 更新延迟 < 100ms

2. **可靠性要求**：
   - 更新操作有错误处理
   - 文件损坏时有备份恢复

## 验收标准

- [ ] 莉莉可以使用简化API更新self prompt（同一个tool支持多种模式：追加/替换/前置）
- [ ] 文件不存在时输出warning日志
- [ ] 向后兼容现有使用方式
